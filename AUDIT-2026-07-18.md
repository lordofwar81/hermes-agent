# Hermes Audit — 2026-07-18

**Scope:** Institutional-grade audit of the Hermes agent codebase (2,630 Python files, ~1.2M LOC). Deep line-by-line on critical paths (agent loop, routing, tools, memory, platforms, gateway, cron); load-level on the long tail. Every finding execution-verified (command + output), not asserted from docs.

**Safety net:** `hermes-agent/` gitignored from parent repo — initialized a local git repo (`hermes-agent/.git`) as a safety baseline before any change. Snapshot at `~/.hermes-snapshots/hermes-agent-20260718-222807.tar.zst`.

---

## Root cause of recurring breakage (now understood)

Not one bug — a recurring **anti-pattern**: "graceful degradation" flags that cache failure states forever, with no recovery path. Three of the worst defects (H-2, H-1, H-3) were the same shape: a transient condition becomes permanent and silent. Combined with config drift that made every audit report noise, and pre-commit hooks that blocked every commit, the system *looked* fixed after each pass but silently rotted between them.
